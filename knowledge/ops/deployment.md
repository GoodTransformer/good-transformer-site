# Good Transformer - Deployment

---

## Overview

The site deploys automatically to GitHub Pages when code is pushed to the `main` branch.

| Property | Value |
|---|---|
| Hosting | GitHub Pages |
| Domain | `goodtransformer.ai` |
| Build | Next.js static export (`npm run build`) |
| Trigger | Push to `main` branch |
| Workflow | `.github/workflows/deploy-pages.yml` |

---

## Deploy process

1. Push code to `main`
2. GitHub Actions workflow runs:
   - Installs dependencies (`npm ci`)
   - Injects environment variables from GitHub Secrets
   - Runs `npm run build` → produces `/out` directory
   - Deploys `/out` to GitHub Pages
3. Site is live at `goodtransformer.ai` within ~2 minutes

---

## GitHub Secrets

The following secrets should be set in the GitHub repository settings. The booking forms and calendar redirects need the required public endpoints; the email fallback is optional.

| Secret name | Description |
|---|---|
| `NEXT_PUBLIC_FORMSPREE_PERSONAL_ENDPOINT` | Formspree endpoint for personal lesson booking form |
| `NEXT_PUBLIC_FORMSPREE_BOOKING_ENDPOINT` | Formspree endpoint for business brief form |
| `NEXT_PUBLIC_CLARITY_CALL_URL` | Calendar booking URL (e.g. Calendly, Cal.com) |
| `NEXT_PUBLIC_PERSONAL_LESSON_URL` | Calendar booking URL for personal AI lessons |
| `NEXT_PUBLIC_BOOKING_BRIEF_EMAIL` | Optional manual fallback email address for intake summaries |

If secrets are not set, `.env.production` provides fallback values (may be empty/placeholder).

---

## Custom domain

The site uses a custom domain (`goodtransformer.ai`) - no `basePath` or `assetPrefix` is needed in `next.config.ts`.

A `CNAME` file in the `public/` directory (or auto-generated by GitHub Pages settings) points to the custom domain.

---

## No manual deploys needed

Do not manually upload files or run `gh-pages` commands. The GitHub Actions workflow handles everything. If you need to trigger a deploy without a code change, push an empty commit:

```bash
git commit --allow-empty -m "trigger deploy"
git push origin main
```

---

## Rollback

To roll back to a previous version, revert the relevant commit and push to `main`. GitHub Actions will rebuild and redeploy the previous state.
